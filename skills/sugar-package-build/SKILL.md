## When to use this skill

Use this skill any time a package needs its own `pack.php`. EVERY MLP in this repo includes a self-contained pack.php that, when run, produces a Module Loader-installable zip in `releases/`. Pack.php discovers files dynamically — never hardcode file lists. For the broader picture of what installdefs sections exist (beans, copy, relationships, vardefs, layoutdefs, sidecar, language), see `[[sugar-mlp-anatomy]]`.

# Pack.php Generation Prompt for Sugar MLP Generator

## Purpose
Every SugarCRM Module Loadable Package (MLP) generated by this system MUST include a proper, executable `pack.php` script that:
- Reads version information from the command-line argument or a `version` file
- Creates a `.zip` file in the `releases/` directory
- Dynamically discovers all files in the `src/` directory tree
- Automatically populates the `installdefs['copy']` array with proper file mappings
- Generates a `manifest.php` file inside the zip with correct SugarCRM metadata
- Can be executed directly: `php pack.php [version]`

## Pack.php MUST NOT Be
- A static PHP return statement (Not a simple array definition)
- A static manifest definition (Not a factory function)
- Missing file discovery logic (Must recursively scan `src/`)
- Missing `ZipArchive` usage (Must actually create zip files)
- Missing dynamic `installdefs['copy']` population (Must auto-generate from src/ contents)
- Missing version handling (Must accept CLI argument or read `version` file)

## Pack.php MUST Do (Correct Implementation)
1. **Read Version**:
   - Check for command-line argument: `$argv[1]`
   - Fall back to `version` file if no argument provided
   - Die with usage message if version cannot be determined

2. **Setup Release Directory**:
   - Ensure `releases/` directory exists (create if necessary)
   - Generate unique zip filename: `sugarcrm-{packageID}-{version}.zip`
   - Verify zip doesn't already exist (prevent accidental overwrites)

3. **Create Manifest Array**:
   - `id`: Package ID (must match packageID variable)
   - `name`: Human-readable label
   - `description`: Feature description
   - `version`: From version file or CLI argument
   - `author`: 'SugarCRM, Inc.'
   - `is_uninstallable`: 'true' (string, not boolean)
   - `published_date`: `date("Y-m-d H:i:s")`
   - `type`: 'module'
   - `acceptable_sugar_versions`:
     - `exact_matches`: Array (empty or specific versions)
     - `regex_matches`: Array (e.g., '(26|25|14)\\..*$')
   - `acceptable_sugar_flavors`: Array (valid: 'ENT', 'ULT', 'PRO', 'TEAM')

4. **Create installdefs Array**:
   - `beans`: Array (typically empty)
   - `id`: Package ID
   - `copy`: Will be populated dynamically in step 6

5. **Create ZipArchive**:
   - Use PHP's `ZipArchive` class
   - Open zip file with `ZipArchive::CREATE` flag
   - Add all files from `src/` recursively

6. **Discover and Add Files**:
   - Use `RecursiveDirectoryIterator` to find all files under `src/`
   - For each file:
     - Get real path: `$file->getRealPath()`
     - Calculate relative path: `'src' . str_replace($basePath, '', $fileReal)`
     - Add to zip: `$zip->addFile($fileReal, $fileRelative)`
     - Add to `installdefs['copy']`: `from => '<basepath>/' . $fileRelative`, `to => preg_replace('/^src[\/\\\](.*)/', '$1', $fileRelative)`

7. **Generate and Add Manifest.php**:
   - Use `var_export()` for arrays
   - Format: `<?php\n$manifest = ...; \n$installdefs = ...;\n`
   - Add to zip root: `$zip->addFromString('manifest.php', $manifestContent)`

8. **Close and Report**:
   - Close zip file: `$zip->close()`
   - Output success message with zip path
   - Exit with status 0

## Validation Checklist
- [ ] Pack.php is executable (shebang `#!/usr/bin/env php`)
- [ ] Reads version from argv[1] or `version` file
- [ ] Creates `releases/` directory if needed
- [ ] Uses `ZipArchive` to create zip file
- [ ] Recursively scans `src/` directory
- [ ] Populates `installdefs['copy']` dynamically
- [ ] Generates `manifest.php` with all required fields
- [ ] Manifest has `regex_matches` in `acceptable_sugar_versions`
- [ ] Manifest has valid `acceptable_sugar_flavors`
- [ ] Zip filename follows pattern: `sugarcrm-{packageID}-{version}.zip`
- [ ] Script exits with status 0 on success
- [ ] All files under `src/` are added to the zip

## Common Mistakes to Avoid
- Using `file_get_contents()` instead of `ZipArchive`
- Hardcoding file lists instead of using `RecursiveDirectoryIterator`
- Returning an array instead of creating a zip file
- Forgetting the shebang line
- Using `ExternalResourceClient::getInstance()` (it doesn't exist)
- Not generating `manifest.php` inside the zip
- Using 'PRO' flavor when package is 'ENT' only
- Using version as integer instead of string in manifest

## Integration with Feature Generator
When generating a complete MLP package:
1. Generate pack.php first (using this template as reference)
2. Generate version file with initial version (e.g., '1.0.0')
3. Generate src/ directory structure with Sugar files
4. Generate releases/.keep file to preserve directory in version control
5. All files output atomically as raw file entries (no interactive steps)
